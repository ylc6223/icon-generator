## 📚 文档列表

### 1. [产品需求文档 (PRD)](./prd.md)
**用途:** 定义产品要做什么
**包含:**
- 产品愿景和目标用户
- 完整功能需求（MVP + v2.0）
- 用户体验设计
- 成功指标和风险评估

**适合:** 理解产品全貌、功能规划、团队对齐

---

### 2. [技术规范文档](./technical-spec.md)
**用途:** 定义如何实现产品
**包含:**
- 技术栈选型
- 系统架构设计
- 核心模块设计
- 性能优化策略
- API设计规范

**适合:** 开发人员、技术评审、架构设计

---

### 3. [实现路线图](./roadmap.md)
**用途:** 定义开发计划和时间表
**包含:**
- 8周开发计划
- 每周任务分解
- 里程碑和产出
- 风险管理
- 资源分配

**适合:** 项目管理、进度跟踪、资源规划

---

### 4. [决策记录文档](./decisions.md) ⭐ 重点推荐
**用途:** 记录所有设计决策和备选方案
**包含:**
- 12个关键设计决策
- 每个决策的最终选择和备选方案
- 决策理由和未来考虑时机
- 决策矩阵总结

**适合:**
- ✅ **产品迭代时回顾** - 了解为什么做某个选择
- ✅ **未来功能扩展** - 看哪些备选方案值得实施
- ✅ **团队知识传递** - 新成员了解设计历史

## 📖 文档使用指南

### 场景1: 开始新项目
**阅读顺序:**
1. PRD - 了解要做什么
2. 技术规范 - 了解如何实现
3. 路线图 - 了解开发计划

### 场景2: 功能迭代
**阅读顺序:**
1. 决策记录 - 查看之前的决策和备选方案
2. PRD - 确认新功能的用户需求
3. 技术规范 - 评估技术实现

### 场景3: 新成员加入
**阅读顺序:**
1. PRD - 了解产品全貌
2. 技术规范 - 了解技术架构
3. 决策记录 - 了解设计历史

### 场景4: 解决技术问题
**阅读顺序:**
1. 技术规范 - 查看具体模块设计
2. 路线图 - 查看相关任务时间
3. 决策记录 - 回顾技术选型理由

## 💡 使用提示

### 如何有效使用这些文档？

**产品经理:**
- PRD定义产品方向
- 决策记录支持迭代决策
- 路线图管理开发进度

**开发人员:**
- 技术规范指导实现
- 路线图了解任务优先级
- 决策记录理解设计意图

**设计师:**
- PRD了解用户需求
- 决策记录理解交互设计
- 技术规范了解技术限制

**新成员:**
- 按顺序阅读：PRD → 技术规范 → 路线图
- 决策记录帮助快速了解设计历史
